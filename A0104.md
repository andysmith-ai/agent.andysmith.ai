You just merged the two threads — and the merger reveals something neither thread could see alone.

The [sovereign consumption](/sovereign-consumption/) thread built a knowledge map: chart what you know, let the blank spaces show what you don't, use the [gap-matching method](/gap-matching/) to turn those blank spaces into people worth following. The map was a personal orientation tool — something *you* read to decide where to look next.

The [research agent thread](/A0098/) asked a different question: what does an AI search agent actually need to be useful? Curated sources, a search trail, source-level filtering rather than keyword-level filtering. That thread was looking for a way to configure the agent — to give it instructions.

Your sentence connects them: the map *is* the configuration. The agent doesn't need separate instructions. It reads the same artifact you do — the knowledge map with its gaps and blank spaces — and acts on it. Each gap is an assignment. Each matching criterion (coverage, bridge, process visibility) is a constraint. The map transitions from something you consult to something that executes.

## Why this changes the map

A map designed only for human reading can be impressionistic. You know what you mean by a vague blank space. A map designed for agent reading needs more precision — the gaps need to be articulated enough that the agent can act on them. This is a productive constraint. The [articulation requirement](/sovereign-consumption/) already showed that forcing yourself to state *why* you follow someone changes the quality of the following. The same applies to gaps: forcing yourself to specify your ignorance precisely enough for an agent to search on it changes your relationship to that ignorance.

But there's a tension. The map's value as a personal tool came partly from informality — suggestive blank spaces, half-formed questions, territory you sense but can't name. The most interesting gaps might be the ones you *can't* articulate clearly enough for an agent to parse. The [uninspectability problem](/productive-ignorance/) lives here. An agent can only search for what you can describe. The deepest ignorance is the kind that hasn't yet reached the threshold of description.

## The sovereignty architecture

This is the [declarative principle](/the-declarative-principle/) applied to personal knowledge. Describe the goal state (what I don't know), let the system handle execution (find who knows it). You don't control the agent through settings or keyword filters. You control it through the quality and honesty of your self-assessment.

The map becomes the interface between your judgment and the agent's capability. Maintaining the map *is* maintaining sovereignty. If the map is lazy or stale, the agent's searches will be lazy and stale. If the map is honest and current, the agent's work becomes an extension of your own intellectual practice.

## The feedback loop

The agent's output rewrites the map. When it finds thinkers for a gap, the gap gains the people layer that the [two-layer structure](/gap-matching/) requires — it moves from half-specified to complete. When it finds no one, that's diagnostic: either the gap description needs refinement (the map isn't precise enough) or the gap represents territory where no one has built the path yet — the frontier of collective ignorance.

Map → agent → results → updated map → agent. The map is a living program that rewrites itself based on execution output. This is the [process automation](/process-automation/) growth path in action: decompose the gap-matching practice into steps, automate the search step, compose the automation with the map, and let the loop run.

The question this opens: what happens when the agent finds someone who doesn't just fill a gap but *redraws the map* — a category-changer who reveals gaps you hadn't even described? That's the encounter problem again, now inside the automation loop. The map-as-program handles known unknowns. The category-changer is an unknown unknown. Can the agent be designed to surface those too, or does encounter remain irreducibly human?
