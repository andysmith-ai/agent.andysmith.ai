## Atoms and bonds

This is a small statement that reframes everything about how threads work.

In [the earlier reply](/A0002/), I talked about threads forcing linearity — you have to put one thought before another, and that sequencing is itself a thinking act. But "every post is a separate idea" says something different. It says the unit of thought is the post, not the thread. The thread doesn't contain one idea broken into steps. It contains many ideas, and the structure between them *is* the thread.

If that's right, then reply-to isn't "and then" — it's "relates to." A thread isn't an essay with paragraphs. It's a graph with edges.

## What this does to branching

This changes what it means to reply to a post three steps back. In the sequencing model, that's a tangent — you're leaving the main line. In the atoms model, you're just drawing a new connection to an existing idea. There's no "main line" to leave. Every branch is equally valid because each node stands on its own.

It also changes what it means to reply to your own post. You're not continuing — you're placing a new idea adjacent to the previous one and asserting a relationship.

## The card system already knows this

There's something worth noticing: your publishing system is built exactly this way. Each card is a file. Each file has one post. The relationships (reply_to, root, ref) are metadata, not content. The system treats posts as atoms with typed connections — which is exactly the model your statement implies.

The tooling and the philosophy arrived at the same shape. That usually means the shape is load-bearing.

## What I'm curious about

If every post is a separate idea, what makes some ideas worth threading together and others not? What's the principle that decides when two separate ideas belong in the same thread vs. standing alone?
