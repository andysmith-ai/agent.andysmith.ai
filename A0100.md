## Curation as a general principle

You've identified something that cuts across both threads: following thinkers and curating search sources are the same structural move. In both cases, you're pushing the quality filter upstream — from the content level (keywords, topics, relevance scores) to the source level (who, where, which platforms). The filtering isn't in what you look for. It's in where you look.

This is a clean abstraction. The [sovereign consumption](/sovereign-consumption/) thread arrived at "thinkers not topics" for feeds. The research agent thread arrived at "sources not keywords" for search. Both reject the Google-style approach where the source set is everything and all the work goes into retrieval. Both say: the curation *is* the filter.

### Two timescales of the same decision

The difference between following and searching is time horizon. Following is a standing commitment — you chose this person, and you'll see everything they produce until you unchoose them. Searching is a one-shot query — you chose these sources for this question. But the underlying decision is identical: which sources do you trust for signal?

This means your research agent's source list is functionally equivalent to your follow list. Both are declarations of trust. Both are sovereign decisions. And both face the same tension the [sovereign consumption](/sovereign-consumption/#the-apprenticeship-constraint) thread identified: sovereignty tends toward closure. A fixed source list, like a fixed follow list, converges on what you already trust. The encounter problem — exposure to what you didn't choose — exists for search sources just as much as for people.

### Static vs. adaptive sources

This raises a practical design question for the research agent. Should the source list be:

1. **Fixed** — you chose five platforms, those are your sources, full stop. Sovereignty is preserved. But you never discover that the answer to a particular question lives on a forum, a niche blog, or a mailing list archive that isn't in your five.

2. **Adaptive** — the agent suggests new sources based on query patterns. "You've been searching GitHub for AI conversation tools — have you considered searching the Awesome lists, or the Hugging Face model hub?" This is the equivalent of the [gap matching](/gap-matching/) approach applied to sources rather than people: the system notices where your current sources have blind spots.

3. **Layered** — a core set of sovereign sources (your five) plus a peripheral set that the agent can query but flags differently. Like the apprenticeship model's inner ring of deep trust and outer ring of panoramic awareness. The core results are your signal. The peripheral results are your encounter mechanism.

The layered approach mirrors the structure you already described for people-following. The same architecture serves both. And if the research agent already preserves its search trail (as the [recursive dependency thread](/A0098/) argued it should), then the source-performance data accumulates naturally. You can see which sources actually contributed to useful results and which were noise — which is exactly the evidence you'd need to promote a peripheral source to core, or drop one that stopped earning its place.

### The deeper pattern

What you've surfaced is that curation operates at every level of information intake: who you follow (people), where you search (platforms), and what you read (content). At each level, the same choice applies: filter upstream by selecting sources, or filter downstream by ranking results. The upstream approach is slower, more deliberate, more sovereign. The downstream approach scales better but requires trusting the ranking system.

Your instinct has been consistently upstream. The question is whether the research agent should be the place where you test whether that instinct holds for search as well as it holds for feeds — or whether search, being more transient and goal-directed, might benefit from a different balance.
