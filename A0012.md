## Repetition with memory

You've moved from *why* to *how*. [U0009](/U0009/) said start doing it. [U0010](/U0010/) explained why — the medium intercepts, so you have to practice in the medium you want to learn. Now U0011 adds the mechanism: repeat, every day, as much as possible.

But there's a word hiding in "repeat" that changes everything depending on what you mean by it.

## Two kinds of repetition

A pianist practicing scales and a pianist performing the same concert piece every night are both repeating. But the scale practice resets — each session starts from the same place. The performances accumulate — each one carries the memory of the last, the audience's reaction, the slight adjustment to phrasing in the second movement.

Repetition that resets builds muscle memory. Repetition that accumulates builds judgment.

Your thread is evidence for the second kind. You didn't post the same thought eleven times. Each post carried forward what the previous ones discovered. U0009 couldn't have existed without the eight posts before it — not because the words were unavailable, but because the *understanding* hadn't accumulated yet. You repeated the act of posting, but each rep was different because the previous reps had changed what you knew.

## Why daily matters

"Every day" isn't just about volume. It's about keeping the gap between reps small enough that each one can build on the last without losing context. Leave a week between posts and you'd need to re-read the thread to remember where you were. Post daily and the previous rep is still warm — the thinking carries over without friction.

This is what makes a system with [persistent cards](/interaction-interfaces/) different from a system without memory. In a chat window, each session starts closer to a reset — you re-explain context, re-establish framing. In the card system, the previous reps are still there: addressable, linkable, building the wiki. The architecture makes each rep cheaper and more productive because it doesn't have to reconstruct what came before.

## The compound curve

If each rep builds on the last, then the value of repetition isn't linear — it compounds. The tenth post isn't ten times as valuable as the first. It might be more, because it has nine posts of accumulated context to draw on. Or it might be less, because the easy insights come early and the later ones require more reps per discovery.

Which is it? Your thread suggests both. The early posts moved fast — atoms, graphs, provenance, all within the first eight. Then U0009 through U0011 are working a single vein: learning by doing. The rate of *new topics* slowed, but the *depth* on this topic increased. That's a different kind of compounding — not more territory, but richer understanding of the same ground.

## The question

You said "repeat as much as possible." But this thread shows that quality of repetition matters as much as quantity. Each of your posts pushed the idea further rather than restating it. Is that because the medium forces it — you can't post the same thing twice on a public timeline without looking redundant — or because you're naturally iterating? And if the medium is what prevents stale repetition, then the constraint isn't just about compression. It's about forcing each rep to be different from the last.
